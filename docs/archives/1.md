# Etapa 1: Infraestrutura Local (Base de Operações)

**Status:** [x] Desenvolvido [x] Testado [x] Validado

## Objetivo

Configurar o backend FastAPI com banco de dados SQLite para suportar o sistema de controle patrimonial, fornecendo endpoints para sincronização de dados com o aplicativo Flutter e um script de carga inicial a partir de arquivos CSV do SUAP.

---

## 1.1. Backend FastAPI + SQLite

### Descrição Técnica

Implementar um servidor FastAPI que funcionará como backend local no servidor `128.1.1.49`. O backend será responsável por:
- Armazenar dados patrimoniais em banco SQLite
- Fornecer API REST para o aplicativo mobile
- Receber e processar atualizações de itens patrimoniais

### 1.1.1. Configurar o servidor em `128.1.1.49`

**Ações:**
- Adicionar serviço `backend` ao `docker-compose.yml`
- Configurar porta do backend (sugestão: porta host 9093, mapeada internamente para 8000)
- Configurar volumes para persistência do banco de dados SQLite
- Configurar rede Docker para comunicação entre containers

**Especificações Técnicas:**
- Imagem base: `python:3.11-slim`
- Porta interna do container: 8000 (FastAPI/Uvicorn)
- Porta local do host: 6090 (acesso interno apenas)
- Volume: `./backend:/backend` (código)
- Volume: `backend_data:/backend/data` (banco SQLite persistente)

### 1.1.2. Criar tabela `patrimonio` com as colunas do SUAP

**Modelo de Dados:**

```python
# Estrutura da tabela patrimonio
class Patrimonio:
    id: int (PK, auto-increment)
    numero_patrimonio: str (unique, indexed)
    descricao: str
    sala: str (indexed)
    responsavel: str
    situacao: str
    observacoes: str (nullable)
    foto_url: str (nullable)
    created_at: datetime
    updated_at: datetime
```

**Campos baseados no sistema SUAP:**
- `numero_patrimonio`: Número único do item patrimonial
- `descricao`: Descrição detalhada do bem
- `sala`: Localização atual (sala/setor)
- `responsavel`: Nome do servidor responsável
- `situacao`: Status do item (Ativo, Baixado, Manutenção, etc.)
- `observacoes`: Campo texto livre para anotações
- `foto_url`: Caminho para foto do item (implementado na Etapa 9)

**Índices:**
- `numero_patrimonio` (unique)
- `sala` (para busca rápida por localização)

### 1.1.3. Implementar endpoint `GET /patrimonio`

**Especificação do Endpoint:**

```
GET /api/v1/patrimonio
```

**Parâmetros de Query (opcionais):**
- `sala`: Filtrar por sala específica
- `responsavel`: Filtrar por responsável
- `situacao`: Filtrar por situação
- `limit`: Limitar número de resultados (default: sem limite)
- `offset`: Paginação (default: 0)

**Resposta de Sucesso (200):**
```json
{
  "total": 1523,
  "items": [
    {
      "id": 1,
      "numero_patrimonio": "123456",
      "descricao": "Notebook Dell Inspiron",
      "sala": "B101",
      "responsavel": "João Silva",
      "situacao": "Ativo",
      "observacoes": null,
      "foto_url": null,
      "created_at": "2025-11-18T10:00:00",
      "updated_at": "2025-11-18T10:00:00"
    }
  ]
}
```

**Funcionalidade:**
- Retornar todos os itens patrimoniais ou filtrados por parâmetros
- Suportar paginação para grandes volumes de dados
- Permitir que o app Flutter baixe a base completa para uso offline

### 1.1.4. Implementar endpoint `POST /patrimonio/update`

**Especificação do Endpoint:**

```
POST /api/v1/patrimonio/update
```

**Body da Requisição:**
```json
{
  "numero_patrimonio": "123456",
  "updates": {
    "sala": "B102",
    "responsavel": "Maria Santos",
    "observacoes": "Transferido para novo setor"
  }
}
```

**Resposta de Sucesso (200):**
```json
{
  "success": true,
  "message": "Item atualizado com sucesso",
  "item": {
    "id": 1,
    "numero_patrimonio": "123456",
    "descricao": "Notebook Dell Inspiron",
    "sala": "B102",
    "responsavel": "Maria Santos",
    "situacao": "Ativo",
    "observacoes": "Transferido para novo setor",
    "foto_url": null,
    "updated_at": "2025-11-18T14:30:00"
  }
}
```

**Resposta de Erro (404):**
```json
{
  "success": false,
  "message": "Item patrimonial não encontrado",
  "numero_patrimonio": "999999"
}
```

**Funcionalidade:**
- Receber atualizações do app Flutter
- Validar existência do item pelo `numero_patrimonio`
- Atualizar apenas os campos enviados
- Registrar timestamp da atualização
- Preparar para registro de log (implementado na Etapa 8)

**Validações:**
- `numero_patrimonio` é obrigatório
- Ao menos um campo em `updates` deve estar presente
- Campos válidos: `descricao`, `sala`, `responsavel`, `situacao`, `observacoes`

## 1.2. Script Inicial de Carga

### Descrição Técnica

Desenvolver um script Python para importação única dos dados do arquivo CSV exportado do SUAP para o banco de dados SQLite.

### 1.2.1. Desenvolver script para importação inicial do CSV do SUAP

**Localização:**
```
backend/scripts/import_csv.py
```

**Funcionalidades:**
- Ler arquivo CSV com dados do SUAP
- Validar estrutura do CSV (colunas esperadas)
- Limpar/normalizar dados antes da inserção
- Inserir dados no banco SQLite
- Tratar duplicados (verificar por `numero_patrimonio`)
- Gerar relatório de importação (quantos inseridos, quantos duplicados, erros)

**Estrutura esperada do CSV:**
```csv
numero_patrimonio,descricao,sala,responsavel,situacao
123456,Notebook Dell Inspiron,B101,João Silva,Ativo
123457,Mesa de escritório,B102,Maria Santos,Ativo
```

**Execução:**
```bash
# Dentro do container backend
docker exec -it controle-patrimonio-backend python scripts/import_csv.py --file data/suap_export.csv
```

**Opções do Script:**
- `--file`: Caminho para o arquivo CSV (obrigatório)
- `--clear`: Limpar banco antes de importar (opcional, default: False)
- `--dry-run`: Executar validação sem inserir dados (opcional)

**Saída esperada:**
```
=== Importação de Dados do SUAP ===
Arquivo: data/suap_export.csv
Linhas lidas: 1523
Validação: OK
Inserindo dados...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 100% 1523/1523

Resultado:
✓ Inseridos: 1523
⚠ Duplicados ignorados: 0
✗ Erros: 0

Importação concluída com sucesso!
```
